32 ##How to develop with htScheme
33 htScheme has been designed as an extensible architecture of scheme-like languages, thus new types of tokens as well as parsers could be easily added into this program.

39 class SchemeUnit
40 {
41  public:

49 ####tokenizer.hpp/cpp
50 ```cpp
51 class Tokenizer
52 {
53  public:
54  Tokenizer(const std::vector<std::string>& lines);
55  void split(const std::vector<std::string>& lines);
56  void parse(const std::list<std::string>& rawTokens);
57  std::list<std::string> rawTokens;
58  std::list<Token> tokens;
59  bool complete;
60 };
61 ```
62 Accept lines of program, Then `Tokenizer::Tokenizer` will call `Tokenizer:split` and `Tokernizer::parse` in order.
63 
64 `Tokenizer::split` splits `lines` into several small string pieces stored in `Tokenizer::rawTokens`.
65 For example, `(string-ith "123 34" 2)` will be split into
66 ```
67 (
68 string-ith
69 "123 34"
70 2
71 )
72 ```
73 
74 `Tokenizer::parse` convert `Tokenizer::rawTokens` to `Tokenizer::tokens`.
75 
76 `Token` is defined as followed in `types/all.hpp`:
77 ```cpp
78 struct Token
79 {
80  TokenType tokenType; //enum TokenType {OpPlus, ...}
81  InfoTypes info; //typedef boost::variant<InfoType1, ...> InfoTypes
82 };
83 ```
84 
85 There is an extra variable `Tokenizer::complete` in this class, which represents whether there is no incomplete brackets or quotaion marks. This could be useful in building command-line interpreter.
86 `Tokenizer::complete` can be set by both `Tokenizer::split` and `Tokenizer::parse`.

89 ```cpp
90 class AST
91 {
92  public:

100 Build an AST which could be accessed through `AST.astHead` with `std::list<Token>`.
101 
102 Here is the definition of `ASTNode`:
103 ```cpp
104 struct ASTNode
105 {
106  NodeType type; //enum NodeType {Bracket, Simple};
107  Token token;
108  ASTNode* parent;
109  std::list<ASTNode*> ch;
110 
111  ASTNode* add(const ASTNode& node); //ch.push_back(new ASTNode(node))
112  void remove(); //Recursively remove all its children then clear ch
113 };
114 ```
115 
116 For example, `(+ 2.7 (- 5.6 2.1) 3)` will be converted to the following AST:
117 ```
118 Type:0 Token.info:0 TokenType:0 //astHead
119 +----Type:Bracket Token.info:0 TokenType:0
120  +---Type:Simple Token.info:0 TokenType:OpPlus
121  |---Type:Simple Token.info:2.7 TokenType:Float
122  |---Type:Bracket Token.info:0 TokenType:0
123  | +---Type:Simple Token.info:0 TokenType:OpMinus
124  | |---Type:Simple Token.info:5.6 TokenType:Float
125  | |---Type:Simple Token.info:2.1 TokenType:Float
126  |---Type:Simple Token.info:3 TokenType:Float
127 ```

129 ####parsers.hpp
130 The main part of `parsers.hpp` is in `parsers/all.hpp`
131 
132 ```cpp
133 class ParsersHelper
134 {
135  ParsersHelper();
136  void parse(ASTNode& astnode);
137 };
138 ```
139 Provide a reference of `ASTNode` to an instance of `ParsersHelper::parse`, then `ParsersHelper` will call according `xxxASTParser::parse(ASTNode& parent, ParsersHelper& helper)` to recursively calculate the result of a subtree of AST with its root as `astnode`.
140 After `ParsersHelper::parse`, `astnode.type` will become `Simple`.
141 If `astnode` is already a `Simple` node, nothing will be done.
142 
143 The parsed version of the above AST is (by calling `parse( **ast.headNode.ch.begin() )`:
144 ```
145 Type:1 Token.info:0 TokenType:0 //headNode
146 +----Type:Simple Token.info:9.2 TokenType:Float
147 ```
148 
149 **Warning** --DO NOT-- directly call `helper.parse(*ch[xx])` in `xxxASTParser::parse(ASTNode& parent, ParsersHelper& helper)` ! Instead, you should copy construct a new `ParsersHelper` to parse its children.

152 ###Add your own Token Parser
153 
154 In this section, we will try to add a new `Rational` type of token.
155 
156 ####Step1: Register the Rational Type
157 Open `types/arch.hpp`, add `Rational` to `enum TokenType{ ... , Rational }`
158 
159 ####Step2: Register the Rational Parser
160  - Open `types/all.hpp`, add `RationalParser` to `#define PARSERS_TUPLE (..., RationalParser)`
161 
162  - `#include "rational.hpp"`
163 
164 ####Step3: Write the Header
165  - Create `types/rational.hpp`, then include your declaration of your `RationalType` and `"arch.hpp"`
166  - Use the macro in `arch.hpp` to generate the declaration of your `RationalParser` :
167  ```
168  PARSER_DECLARATION(RationalParser, Rational, RationalType)
169  ```
170 
171 ####Step4: Implement the Parser
172  - Create `types/rational.cpp`, then `#include "rational.hpp" ` and implement your `RationalType` in it.
173  - Implement `bool RationalParser::judge(const std::string& token)` which returns whether a token is a rational token
174  and
175  `RationalParser::InfoType RationalParser::get(const std::string& token)` which converts the token to `RationalParser::InfoType`(aka `RationalType`)
176  - `const TokenType RationalParser::type = Rational; `
